# Safety Checklist

Use this checklist before starting and before finishing any MCC task.

## Before Work

- Architecture read.
- Current status read.
- Task queue read.
- Assigned task confirmed.
- Scope boundaries confirmed.
- Required inputs confirmed.

## During Work

- No secrets written to files.
- No live webhook or trade action.
- No unapproved package install.
- No unapproved core file edits.
- Missing data marked as `WAITING_FOR_USER`.

## After Work

- Report created.
- Status files updated.
- Task history updated.
- Failures and repeated failure patterns noted.
- Next action is clear.
