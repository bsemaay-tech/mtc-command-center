# SDLC Process

MTC Command Center should use a professional software development process with architecture and safety boundaries established before implementation.

1. Architecture / Single Source of Truth
2. Requirements
3. Data contracts
4. Folder structure
5. MVP roadmap
6. Design docs
7. Implementation planning
8. Read-only prototype
9. Test strategy
10. Acceptance criteria
11. Controlled implementation
12. Verification
13. Handoff
14. Maintenance

## Operating Principle

Every phase should produce documents or reports first, then controlled implementation only after the user approves the scope.
